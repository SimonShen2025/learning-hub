## 中文短总结

本节介绍 AWS 监控的重要性。监控确保应用正常运行，涵盖日志（Logs）、指标（Metrics）、追踪（Tracing）和审计（Audit）。作为开发者/架构师，在 AWS 上部署应用时必须启用监控，这是最佳实践。

## 中文长总结

本节引入 AWS 监控主题，强调监控在云应用中的关键作用。

**监控四大维度：**
- **日志（Logs）**：记录应用和基础设施的日志
- **指标（Metrics）**：监控性能数据（CPU、网络等）
- **追踪（Tracing）**：分布式追踪
- **审计（Audit）**：记录谁在 AWS 基础设施中做了什么操作

**核心观点：** 部署应用到 AWS 时，务必启用监控，否则问题发生时无法排查。

## 考试要点

- 监控是 AWS 最佳实践的核心组成部分
- 本节涵盖 CloudWatch（Metrics/Logs/Alarms）、CloudTrail（审计）、Config（合规）

## English Short Summary

Introduction to AWS monitoring importance: ensuring applications run correctly through logs, metrics, tracing, and auditing. Never deploy applications to AWS without enabling monitoring — it's essential for troubleshooting and operations.
