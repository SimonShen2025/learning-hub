## 中文短总结

考试实用建议：①**练习 = 完美** 不要急于考试，手动实操是最好的学习方式②如果觉得课程内容多，可以**再过一遍**③**排除法**：先排除明显错误答案，缩小到 1-2 个选项④**不要过度思考**：太复杂的答案通常是错的⑤不确定的题目先答再用 **Flag** 标记，回来复查⑥推荐阅读 3 篇白皮书（AWS Best Practices / Well-Architected / Disaster Recovery）⑦不确定某服务时看 **AWS FAQ 页面**（如 amazon.com/vpc/faqs）⑧参与 AWS 社区（Q&A / 论坛 / meetup / re:Invent 视频）。

## 中文长总结

通过考试的核心策略：

| 策略 | 说明 |
|------|------|
| 充分练习 | 不要急，手动实操 |
| 重复学习 | 觉得多就再过一遍课程 |
| **排除法** | 先排除确定错误的选项 |
| 不过度思考 | 太复杂 = 可能错误 |
| Flag 标记 | 不确定→先答→Flag→回头复查 |
| 白皮书 | 3 篇推荐阅读 |
| FAQ | 服务 FAQ = 考试题目来源 |
| 社区 | Q&A / 论坛 / meetup / re:Invent |

**推荐白皮书：**
1. Architecting for the Cloud: AWS Best Practices
2. AWS Well-Architected Framework
3. AWS Disaster Recovery

## 考试要点

- 排除法是最有效的答题策略
- 不过度思考 — 复杂答案通常是错的
- Flag 功能用于标记不确定的题目
- AWS Service FAQ = 考试题目重要来源
- 推荐 3 篇白皮书

## English Short Summary

Exam tips: (1) **Practice makes perfect** — don't rush, hands-on experience is key; (2) Review course again if overwhelmed; (3) **Proceed by elimination** — rule out obviously wrong answers; (4) **Don't overthink** — overly complicated answers are likely wrong; (5) Use **Flag** feature for uncertain questions; (6) Read 3 whitepapers (AWS Best Practices/Well-Architected/Disaster Recovery); (7) Check **AWS Service FAQs** (e.g., amazon.com/vpc/faqs) — exam questions originate here; (8) Join AWS community (Q&A/forums/meetups/re:Invent videos).
