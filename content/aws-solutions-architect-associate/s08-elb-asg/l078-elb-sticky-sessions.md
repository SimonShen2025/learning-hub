## 中文短总结

Sticky Sessions（会话粘性/Session Affinity）使同一客户端的后续请求始终转发到同一后端实例，通过 Cookie 实现。支持 CLB、ALB 和 NLB。Cookie 分两类：应用级 Cookie（自定义 Cookie 由应用生成，或 AWSALBAPP 由 ALB 生成）和持续时间 Cookie（AWSALB/AWSELB 由负载均衡器生成，按设定时长过期）。粘性可在 Target Group 属性中启用，用于保持用户会话（如登录状态），但可能导致负载不均。

## 中文长总结

**Sticky Sessions 概念：**
- 同一客户端的请求始终路由到同一后端 EC2 实例
- 通过 Cookie 机制实现，Cookie 有过期时间
- 过期后客户端可能被重定向到不同实例

**支持的负载均衡器：** CLB、ALB、NLB

**Cookie 类型：**
1. **应用级 Cookie（Application-based）：**
   - 自定义 Cookie：由应用自身生成，可包含自定义属性
   - 应用 Cookie：由 ALB 生成，名为 `AWSALBAPP`
   - 不能使用保留名：AWSALB、AWSALBAPP、AWSALBTG
2. **持续时间 Cookie（Duration-based）：**
   - 由负载均衡器生成
   - ALB 名为 `AWSALB`，CLB 名为 `AWSELB`
   - 过期时长由负载均衡器设置

**使用场景：** 保持用户会话数据（如登录状态）

**注意事项：** 启用粘性可能导致后端实例负载不均衡

**配置方法：** Target Group → Actions → Edit Attributes → Stickiness → 选择 Cookie 类型和时长

## 考试要点

- Sticky Sessions 通过 Cookie 实现，支持 CLB/ALB/NLB
- 两种 Cookie 类型：Application-based 和 Duration-based
- ALB 保留 Cookie 名：AWSALB、AWSALBAPP、AWSALBTG
- 用于保持用户登录等会话状态，但可能导致负载不均

## English Short Summary

Sticky Sessions (Session Affinity) ensure the same client always reaches the same backend instance via cookies. Supported on CLB, ALB, and NLB. Two cookie types: application-based (custom by app or AWSALBAPP by ALB) and duration-based (AWSALB/AWSELB generated by the load balancer with configurable expiry). Configured at the target group level. Useful for maintaining session data like login state, but may cause load imbalance.
