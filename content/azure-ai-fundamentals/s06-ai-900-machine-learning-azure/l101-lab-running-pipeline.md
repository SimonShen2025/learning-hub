## 中文短总结

运行 Pipeline：Configure and Submit → 创建 Experiment（实验）→ 选择 Compute Instance（非 Cluster）→ Submit。Pipeline Job 在计算实例上执行。大型 Pipeline 用 Cluster 更快但更贵。成本组成：专业知识 + 数据准备时间 + 计算基础设施。模型越大/数据越多 → 参数越多 → 需要更大计算能力 → 费用越高。

## English Short Summary

Run Pipeline: Configure and Submit → create Experiment → select Compute Instance → Submit. Pipeline Job executes on compute machine. Larger pipelines use Clusters (faster but costlier). Costs: expertise + data prep time + compute infrastructure. Bigger models/more data → more parameters → more compute needed → higher cost.
