## 中文短总结

ML 生命周期流程：1) 定义业务用例（为什么建模？解决什么问题？）→ 2) 准备数据（识别数据源、清洗、特征工程）→ 3) 选择算法 → 4) 训练模型 → 5) 评估模型性能 → 6) 部署模型到生产环境 → 7) 监控和维护。ML 模型本质是文件/数学模型，需要计算基础设施和投资（时间/金钱/数据）。

## 考试要点

- ML 工作流：业务用例 → 数据准备 → 算法选择 → 训练 → 评估 → 部署 → 监控
- 模型 = 文件（数学表示）
- 两个核心要素：数据集 + ML 算法
- 需要计算基础设施（成本）
- 先定义业务问题再开始建模

## English Short Summary

ML workflow: 1) Define business use case 2) Prepare data (identify sources, clean, engineer features) 3) Choose algorithm 4) Train model 5) Evaluate performance 6) Deploy to production 7) Monitor & maintain. ML model is essentially a file (mathematical representation). Requires compute infrastructure investment (time, money, data).
